# CRM layer

Этот раздел не трогает рабочую диагностику в `doctor/index.html`.

План:

1. `doctor/index.html` — оставить как рабочую диагностику с атласом.
2. `crm/index.html` — отдельная карточка пациента.
3. Из CRM открывать диагностику ссылкой на `../doctor/`.
4. Позже добавить передачу `patient_id` и сохранение результата в Supabase.

Так мы больше не ломаем рабочую диагностику.
